트레이닝 테이블에 "안내선 필터" 이미지 컬럼(또는 테이블) 생성

치료사 스케쥴 = 캘린더로 표현
하루일자를 클릭하면, 해당 날짜의 치료목록이 나온다.

치료 입력할 때, 이름으로 검색하고, 드롭박스로 이름-아이디식으로 보여준다.

============================================================

DB에 컬럼을 추가하여 FE의 작업을 편하게 할까 vs 이미 있는 컬럼을 반환해주면 FE에서 코드를 작성하여 충분히 해결할 수 있는데 컬럼을 더 추가해야 할까
  => DB에 컬럼을 추가하여 FE에서 수행하는 프로그램의 시간복잡도를 조금이라도 많이 줄이고 FE 분들이 처리를 펀하게 할 수 있도록 하자. FE 분들이 디자인도 많이 하는데, 코드 작성의 부담은 줄여주자.

REST API 에서 비슷한 역할을 하는 기능이 있을 때
URI를 하나로 하고, 퀴리스트링의 값만 바꿔서 코드의 중복을 줄이기 vs URI를 3개로 나눠서 각각의 코드를 작성하고 직관적으로 표현하기
  => 직관적으로 표현하여 FE, BE 둘 다 코드에 대한 이해를 쉽게 하자.

영어 단어의 통일
  => 진행 중인 것과 완료된 것에 대한 URI를 짤 때, URI 이름을 present 와 complete로 함
  => present로 했으면 past로 하자. complete로 할거면 uncomplete로 하자
